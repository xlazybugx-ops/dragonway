# UX_AUDIT.md — аудит всех экранов

Метод: для каждого экрана — главная задача, частое действие, отвлекающее, скрываемое, дубли, длина
пути, перегруз, «мёртвые» данные. В конце — узкие места.

---

## Обзор экранов (до реворка)

Навигация была **хаб-центричной**: главный экран-иллюстрация с 8 кликабельными строениями + ряд
из 6 служебных кнопок; на каждый второстепенный экран добавлялась кнопка «🏠 На главную». Единой
нижней навигации не было — любой переход между разделами шёл через хаб (2 нажатия вместо 1).

| Экран | Главная задача | Частое действие |
|---|---|---|
| Хаб | попасть в нужный раздел | тап по строению |
| Логово | смотреть/качать драконов | выбрать дракона, покормить |
| Гнездо | высиживать яйца | согреть/вылупить |
| Странствие | выбрать биом и лететь | «Странствовать →» |
| Турнир | бой на арене | выбрать бойца |
| Гнездилище | скрещивание | выбрать пару |
| Кузница | ковать реликвии | улучшить |
| Шпиль | дерево навыков | вложить очко |
| Кодекс | коллекция/вехи | листать |

## Поэкранно: проблемы

**Хаб.** Отвлекает: ряд из 6 кнопок (🎁🛒🎨🔊💾♿) смешивает игровое и служебное. Служебное
(звук/сейв/доступность) игрок читает крайне редко, но оно занимает первый экран. Путь до любого
раздела — через хаб. → служебное вынести, дать прямую навигацию.

**Логово.** Задача размыта: тут и роутинг, и роскошный свиток дракона с длинным лором,
характеристиками, генами, реликвиями, природой — всё раскрыто одновременно. Много текста, который
читается редко. → одна главная задача (команда), детали — по нажатию (уже частично: карусель +
деталь; лор длинный).

**Гнездо.** В целом сфокусировано; вводный абзац длинный. → приглушить текст.

**Странствие (выбор биома).** Строка биома несёт: ярус, имя, 🪙диапазон, 🥚%, 🗺️%, статус —
плотно, но читаемо. Экономические числа (🪙диапазон, 🥚%) читаются редко до полёта. → второстепенное
под нажатие.

**Турнир / Гнездилище / Кузница / Шпиль.** Узкие, одноцелевые — ок. Попадание в них — только через
хаб. Длинные пояснения-`lede` в шапках.

**Кодекс.** Три вкладки (Виды/Легенды/Вехи) — уместно. Полные описания видов раскрыты списком. →
в списке главное, детали отдельно.

**Полёт / Аркада (полноэкранные).** HUD уже минимальный (бары/цель/навыки). Нижняя навигация не
должна их перекрывать. → прятать таббар в полноэкранном режиме.

## Узкие места (зафиксировано)

1. **Нет постоянной навигации** — переход раздел→раздел стоит 2 нажатий (через хаб). ❗ главный.
2. **Служебное на первом экране** — звук/сейв/доступность перегружают хаб, читаются редко.
3. **Нет «профиля»** — прогресс игрока, серия входов, коллекция, настройки разбросаны.
4. **Перегрузка деталями** — свиток дракона и списки кодекса показывают всё сразу.
5. **Длинные `lede`-пояснения** в шапке почти каждого экрана — визуальный шум.
6. **Более 3 акцентов на экране** — хаб (8 строений + 6 кнопок) нарушает правило «1 главный + ≤3
   второстепенных».

## Приоритет исправлений

P0: единая нижняя навигация (5 разделов) + экран Профиль (узкие места 1–3).
P1: вынос служебного с хаба; приглушение `lede`; детали по нажатию (4–5).
P2: свести акценты к «1 главный + ≤3» на каждом экране (6) — итеративно.
