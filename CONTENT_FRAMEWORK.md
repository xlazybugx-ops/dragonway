# CONTENT_FRAMEWORK.md — данные отдельно от кода

Цель: игру можно развивать, редактируя таблицы данных, а не логику. Добавление дракона, врага,
яйца, биома, босса, награды — это новая строка в таблице, без правок игровых функций.

---

## 1. Слои архитектуры

```
ДАННЫЕ (что в игре есть)      ЛОГИКА (как это работает)        ПРЕДСТАВЛЕНИЕ (как выглядит)
00-balance.js  — коэффициенты  04-systems.js  — системы          05-ui-base.js — карточки/логово
01-data-core.js — драконы/ходы 07-flight.js   — исследование     06-world.js   — хаб/карта
02-data-content — контент      08-battle.js   — бой              10-spire-codex — кодекс
03-state.js    — схема сейва   08b-arcade.js  — аркада           css/style.css — стиль
```

Правило: файлы данных не знают про DOM; файлы логики читают данные через хелперы-геттеры;
представление читает и данные, и состояние, но не хранит правил баланса.

## 3. Новая таблица: LAIR_LEVELS (ЧАСТЬ 3)

```js
const LAIR_LEVELS=[
  {lvl:1, cap:4,  cost:null},
  {lvl:2, cap:6,  cost:{gold:800,   dust:15}},
  {lvl:3, cap:8,  cost:{gold:2600,  dust:40}},
  {lvl:4, cap:10, cost:{gold:6500,  dust:80,  boss:1}},
  {lvl:5, cap:12, cost:{gold:15000, dust:150, boss:2}},
];
```

Логика (`lairCap`, `lairNext`, `lairUpgradeCheck`, `upgradeLair`, `toggleReserve`) **только читает**
эту таблицу. Изменить кривую роста = отредактировать числа. Добавить 6-й уровень = добавить строку —
UI, проверка стоимости и вместимость подхватят автоматически.

## 4. Контракт «добавить дракона без правки логики»

1. Добавить строку в `SPECIES` (01) — вид появляется в игре, кодексе, разведении, бою.
2. (Опц.) строка в `DRAGON_META` (02) — роль/пассивка/факт.
3. (Опц.) привязать к яйцу: добавить id вида в поле `dragons:[...]` нужного `EGG_CATALOG`.

Ни одной правки в 04/07/08/08b/06 не требуется: бой берёт статы через `makeCombatant`→`statsOf`,
пул видов — через `weightedSpecies`, изображение — через `flySprite`/`sigilHTML`. Все они итерируют
таблицы, а не хардкод-списки.

## 5. Изменённые файлы (ЧАСТИ 1–3)

| Файл | Изменение |
|---|---|
| `02-data-content.js` | + `LAIR_LEVELS` и хелперы логова (декларативно) |

| `07-flight.js` | − разброс яиц по карте (ЧАСТЬ 2) |
| `05-ui-base.js` | панель уровня логова + резерв (представление) |

Совместимость: старые сейвы работают — `lairLevel` по умолчанию 1 (геттеры guard `||1`),
флаг `reserve` отсутствует = дракон активен. `node --check` всех модулей и конкатенации чист.

## 6. Рекомендации для долгосрочного развития

1. Вынести `SPECIES`/`EGG_CATALOG`/`WORLD_BOSSES` в отдельные `data/*.js` (или JSON), подключаемые
   первыми — тогда контент-дизайнер правит только `data/`, инженер — только `js/`.
2. Ввести единый валидатор данных при загрузке (проверка обязательных полей строки) — ошибка в
   таблице ловится сразу, а не падением в бою.
3. Версионировать таблицы (`schemaVersion`) для миграций сейва.
4. Свести все «магические» списки эмодзи-иконок в один `ICONS` реестр.
5. Для локализации — вынести строки в `L10N` словарь по ключам (сейчас текст встр
