# COMBAT_AUDIT.md — аудит боевой системы

В игре два боя: **пошаговый** (арена/турнир, `08-battle.js`, 1×1) и **аркадный** реального времени
(встречи в полёте/логова, `08b-arcade.js`, 1×стая). Аудит обоих по требованиям читаемости.

---

## Пошаговый бой (арена)

| Проверка | До реворка | Оценка |
|---|---|---|
| Читаемость интерфейса | HP/мана барами+числами, оценка урона на приёмах, стихия ▲/▽ | хорошо |
| Отображение HP | бар + `текущее/макс`, обновление в кадре | хорошо |
| Отображение энергии | бар маны + `💧 x/max` | хорошо |
| **Очередь ходов** | **нет индикатора «кто ходит/следующий»** | ❗ пробел |
| **Отображение эффектов** | защита/парир/щит босса/ярость/застыл **не показаны на бойцах** | ❗ пробел |
| Выбор цели | цель одна (соперник), но **не помечена** | ❗ слабо |
| Зоны поражения | одиночный бой — н/д | н/д |
| Массовые/одиночные | всегда одиночная | ок |
| Анимации попаданий | `shake/lunge`, всплывающие числа, вспышка лога | хорошо |
| **Готовность ульты** | кнопка не обновляла статус между ходами (мана-гейт устаревал) | ❗ пробел |

## Аркадный бой (полёт/логова)

| Проверка | До реворка | Оценка |
|---|---|---|
| Читаемость | минимальный HUD: HP/щит/мана бары, счётчик врагов | хорошо |
| HP | бар игрока сверху + мини-бары над врагами | хорошо |
| Энергия | бар маны | хорошо |
| Очередь | реальное время — н/д (телеграфы врагов вместо очереди) | н/д |
| Эффекты | значки 🔥/🟢/❄️ над врагами, кольца щита/ускорения игрока | хорошо |
| Выбор цели | кольцо активной цели + линия атаки | хорошо |
| Зоны поражения | предпросмотр до подтверждения: круг/линия/радиус + счётчик 🎯 (v28) | хорошо |
| **Цепочка (молния)** | **перескоки не предсказывались** | ❗ пробел |
| Массовые/одиночные | площадь подсвечивает всех в радиусе; одиночная — цель+линия | хорошо |
| Анимации | вспышки, частицы, тряска, всплывающие числа | хорошо |
| Обратная связь | fx-зона появляется/гаснет, числа урона | хорошо |

## Узкие места (зафиксировано)

1. Пошаговый: **нет очереди ходов** — игрок не видит, чей ход и кто следующий.
2. Пошаговый: **эффекты невидимы** — защита, парирование, щит/ярость/застывание босса нигде не
   отображались, хотя влияют на решения.
3. Пошаговый: **цель не помечена** — соперник очевиден, но нет явного маркера.
4. Пошаговый: **готовность ульты устаревала** — статус мана-гейта не обновлялся между ходами.
5. Аркада: **цепочка молнии** не показывала будущие перескоки и число целей.

Конус как отдельная зона в игре отсутствует (нет конусных способностей) — предпросмотр покрывает
существующие формы: линия, круг, радиус, цепочка; одиночная — линия+кольцо.
