# PROJECT_STRUCTURE.md — структура репозитория

## Корень (деплой-файлы — НЕ перемещать)
- `index.html` — точка входа; подключает `css/`, `js/`, `images/` относительными путями с `?v=N`.
- `css/style.css` — единая таблица стилей.
- `js/00-balance … 11-save-init` — 13 модулей, строгий порядок загрузки, общий глобальный скоуп.
- `images/` — растровые ассеты (спрайты видов, фоны полёта/логовищ, хаб) + процедурные fallback.
- `.nojekyll`, `.gitignore`, `LICENSE`.

> **Почему runtime-папки в корне.** Деплой — GitHub Pages из корня. Перенос `js/css/images` в
> `/src`,`/assets` потребовал бы переписать все пути в 13 модулях и index.html — рискованно. Решение
> (см. `ai/DECISIONS.md`): держать деплой-структуру плоской, а инженерную — в отдельных папках.

## Инженерные и контентные папки
- `design/` — дизайн-система (арт-дирекшн, гайды, `tokens/tokens.{css,json}`).
- `prompts/` — AI-промпты: 17 категорий (README в каждой) + 13 `MASTER_*` + библиотека/шаблоны.
- `ai/` — постоянный контекст и правила для ИИ (12 документов).
- `docs/` — `audits/`, `reports/`, `design-history/` + финальные отчёты и индексы.
- `tests/`, `tools/` — заготовки для будущих тестов и утилит.
- `_archive/` — устаревшее/эксперименты/бэкапы (не участвует в игре).

## Правила именования
- Папки — строчные (`design`, `prompts`), рантайм — как есть (`js`,`css`,`images`).
- Документы — `UPPER_SNAKE.md`. Ассеты — `snake_case` по `design/ASSET_NAMING.md`.
- Модули — `NN-kebab.js` с числовым префиксом порядка загрузки.
