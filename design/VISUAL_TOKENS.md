# VISUAL_TOKENS.md — библиотека токенов

Единый источник правды. Файлы: `tokens/tokens.css` (CSS-переменные) и `tokens/tokens.json` (зеркало
для инструментов/AI). **Все** будущие компоненты и ассеты используют только эти токены.

## Категории
- **Цвет:** бумага, чернила, небо, материалы, стихии, редкость.
- **Градиенты:** sky, sunset, forest, water, gold, card.
- **Тени:** sm/md/lg (лавандовые), glowSoft, glowLegend, inset-подсветка.
- **Радиусы:** xs 8 · sm 14 · md 20 · lg 28 · xl 36 · pill.
- **Линия:** ширина 2px, тёплый коричневый контур.
- **Типографика:** display (Baloo 2/Fredoka), body (Nunito/Quicksand) + шкала hero/h2/body/caption.
- **Отступы (воздух):** xs 6 · sm 10 · md 16 · lg 24 · xl 36.
- **Движение:** fast 160 · mid 220 · slow 360 мс + единая ease.

## Как использовать
```css
@import 'art/tokens/tokens.css';
.card{ background:var(--ds-grad-card); border-radius:var(--ds-r-lg);
  box-shadow:var(--ds-shadow-md); color:var(--ds-ink); font-family:var(--ds-font-body); }
```
Для генеративного/скриптового пайплайна читать `tokens.json` (стабильные ключи).

## Правила
1. Ни одного «магического» цвета/радиуса/тени в компонентах — только токены.
2. Новый токен добавляется в ОБА файла (css + json) одновременно.
3. Значения меняются централизованно — весь стиль перекрашивается из одного места.
4. Токены версионируются (`_meta.version` в json).

## Связь с игрой
Токены НЕ подключены к текущему билду (чтобы ничего не сломать). При миграции: подключить `tokens.css`
первым, затем переводить компоненты `style.css` на переменные `--ds-*` по одному разделу.
