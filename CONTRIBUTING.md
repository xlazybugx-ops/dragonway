# CONTRIBUTING

## Прежде чем начать
Прочти `ai/PROJECT_CONTEXT.md`, `ai/AI_RULES.md`, `ai/CODING_RULES.md`.

## Правила
- Не менять баланс (`GB`), механики, структуру данных, порядок загрузки без явной задачи.
- Никаких дублей верхнеуровневых имён (общий глобальный скоуп).
- Крупные файлы править точечно, не переписывать целиком.
- Новые поля состояния `S` — с дефолтом и `||`-guard (совместимость сейвов).
- Использовать токены `design/tokens/` для любого UI.

## Проверка перед PR
- `node --check` каждого модуля и конкатенации в порядке загрузки — чисто.
- Нет DEBUG/console.log (кроме `console.warn` в catch).
- Ссылки (js/css/images) целы; при изменении ассетов поднят `?v=N`.
- Обновлены `CHANGELOG.md` и при необходимости `ai/DECISIONS.md`.

## Стиль контента/ассетов
Только по `design/` и `prompts/`. Имена — по `design/ASSET_NAMING.md`.

## Коммиты
Понятные сообщения на русском; одна логическая задача — один коммит.
