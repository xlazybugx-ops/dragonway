# PLAYER_FLOW.md — система «Следующий шаг»

Цель: игрок никогда не теряется. Всегда видит одну ближайшую цель и понимает — **что делать сейчас,
зачем, что получит, что откроется дальше.**

---

## 1. Карточка «Следующий шаг»

На главном экране вместо перегруженной строки целей — одна карточка `nextStepCardHTML`:

```
СЛЕДУЮЩИЙ ШАГ
🐣  Высиди готовое яйцо
    Инкубация завершена
🎁 новый дракон · 🔓 сильнее стая
[ В Гнездо → ]
▾ Все цели
```

- **Что делать** — заголовок (действие).
- **Зачем** — подстрока-причина.
- **Что получишь / что откроется** — строка `🎁 награда · 🔓 разблокировка`.
- **CTA** — кнопка ведёт прямо туда одним нажатием.
- **Все цели** — спойлер с полной инфо-строкой (коллекция, знания, задания и т.д.) — открывается
  отдельно, не засоряя экран.

---

## 2. Приоритет подсказки (`nextStep`)

Движок выбирает единственную важнейшую цель по порядку:

1. 🎁 Готов подарок дня → забрать (серия входов растёт).
2. 🐣 Дозрело яйцо → высидеть (новый дракон).
3. 🏅 Готова веха → забрать награду.
4. 🔓 Близко открытие механики (Кузница@3 / Шпиль@5 / Гнездилище@8) → подрасти на N ур.
5. ✨ Задание дня: выполнить или забрать награду.
6. 🥚 Идёт инкубация → бои/странствия греют яйцо.
7. ⛰️ Ворота биома (ур.60 / ур.100) → цель среднего/позднего этапа.
8. 🏰 Хватает на улучшение логова → улучшить (+вместимость).
9. ☠️ Доступен владыка мира → вызов (уникальное яйцо).
10. 🗺️ Иначе → в странствие (золото, опыт, находки).

Каждый пункт всегда даёт **действие + причину + награду + (где есть) разблокировку**, поэтому пустого
«что делать?» не возникает ни на одном этапе — включая «средний провал» ур.8–60 из аудита.

---

## 3. Меньше шума

Второстепенные данные (проценты коллекции, знания мира, счётчики) убраны с первого плана в спойлер
«Все цели». На виду — одна цель. Это согласуется с реворком логова (одно следующее действие там же).

## 5. Рекомендации

1. Дублировать мини-подсказку следующего шага в топбар (всегда на виду).
2. Подсветка соответствующей вкладки/строения при активной подсказке.
3. Заполнить «средний провал» ур.8–60 промежуточной механикой (мини-открытие ~ур.20–30).
4. A/B длительность стрика и порогов биомов по метрикам удержания.
