# SYSTEM_GRAPH.md — единая карта игровых систем

Цель: связать все механики в единую систему, где каждая усиливает минимум две другие; устранить «мёртвые»
связи интеграцией, а не удалением. Работа в существующей архитектуре — бой/экономика/сюжет/локализация/
структура не тронуты (`node --check` всех модулей чист).

---

## 1. Карта систем (получает → производит → использует → открывает)

| Система | Получает | Производит | Использует | Открывает |
|---|---|---|---|---|
| **Исследование (полёт)** | дракон, регион, стамина | золото, пыль, 🔮осколки, яйца, свитки, сундуки, записи мира, % изучения | дракон (роль/стихия), погода, события | биомы, секреты, встречи, задания-прогресс |
| **Бой (арена/аркада)** | дракон, навыки, стихии | опыт, золото, яйца, инкубация(+), достижения | пассивки, стихии, тайминг, ИИ-роли | уровни, таланты, реванш боссов |
| **Логова (групповой бой)** | стая, ярус биома | яйца, золото, инкубация(+) | стихии, AoE, рывок | прогресс исследования |
| **Драконы** | яйца, опыт, забота | боевую силу, пассивки, роль | стихия/телосложение | исследование быстрее (Следопыт), Кодекс |
| **Яйца** | бои/боссы/исследование/задания/сундуки | драконов, титулы, окрас | редкость, инкубация, 🔮осколки | новые виды, Кодекс Яиц |
| **Боссы** | сильного дракона, стратегию | уникальные яйца, трофеи, ⭐звёзды, легенды, инкубация(+) | фазы, слабости, щит, реванш | биомы-легенды, достижения, Кодекс владык |
| **Экономика** | золото/пыль/🔮осколки/⭐звёзды | улучшения | поэтапное открытие | кузница, кузня, восхождение |
| **Артефакты** | реликвии, золото, пыль | боевые бонусы (fx) | кузница (ковка) | усиление драконов |
| **Опыт/Уровни** | бои, исследование | уровни, таланты | XP-кривая | навыки/ульты (Шпиль), гейтинг механик |
| **Кодекс** | все системы (виды/яйца/мир/боссы/легенды/вехи) | знания, % коллекции | трекеры S.* | достижения, мотивацию |

| **Ежедневные** | вход, действия | золото/пыль/яйца, стрик | фильтр по открытым механикам | подарки, рутинные цели |
| **Логово (дом)** | драконов, заботу | доверие, эмоции, косметику | характеры/настроение | привязку, возвраты |

## 2. Главный игровой цикл (замкнут)

```
Исследование → яйцо/осколки/свиток → (инкубация ускоряется боями/исследованием/осколками)
   → новый дракон (роль/пассивка) → сильнее бой → босс → уникальное яйцо + легенда + достижение
   → Кодекс растёт → новые цели → новый регион → Исследование …
```
Каждый переход подпитывает ≥2 системы: напр. **боссы** дают яйца (эконом. коллекции) + легенды (Кодекс) +
достижения (Вехи) + инкубацию (яйца) + реванш (прогрессия).

## 3. Устранённые слабые места (интеграция, а не удаление) ✅

| Было «повисшим» | Стало связанным |
|---|---|
| **🔮 Осколки** производились (переработка яиц, события мира), но **не тратились** | Новый сток: **«🔮 Согреть»** ускоряет инкубацию яйца (Гнездо). Осколки: исследование/переработка → инкубация → дракон. Связывает исследование ↔ яйца ↔ драконы |
| **Драконы почти не влияли на исследование** | Роль **Следопыт** ускоряет прогресс исследования (×1.5). Драконы ↔ исследование |
| **Достижения были одиночными** | Добавлены **кросс-системные вехи**: Картограф (исследование→100%), Хранитель кладки (8 типов яиц), Знаток мира (8 событий), Двойной триумф (2 босса). Каждая связывает 2–4 системы |
| **Кодекс как разрозненные вкладки** | Агрегатор **«📖 Знания: X%»** на главном экране = виды + яйца + биомы + боссы. Кодекс — центр знаний |
| **Боссы = мешок HP** (ранее) | Дают уникальные яйца, легенды, достижения, реванш; фазы/слабости — стратегия |

(Ранее в этой серии уже связаны: инкубация ← бои/исследование; яйца ← боссы/события/задания; события мира →
Кодекс мира; погода/биом → находки; поэтапное открытие механик по уровню.)

## 4. Новые игровые циклы

1. **Цикл осколков**: исследование/переработка → 🔮 → «согреть» яйцо → быстрее дракон → сильнее исследование.
2. **Цикл знаний**: любая активность → запись в Кодекс → рост «Знаний %» на главном → мотивация к недостающей
   активности (переключение между системами).
3. **Цикл боссов**: прокачка → босс → уникальное яйцо + легенда + достижение + реванш (сложнее/ценнее).
4. **Цикл ролей**: вид дракона (роль) → стиль боя И скорость исследования → выбор дракона под активность.

---

## 5. Переключение активностей каждые 15–20 мин

Инфо-строка главного экрана показывает **ближайшую цель, инкубацию, задания, коллекцию, знания, подарок** —
игрок видит, какая система «просит внимания», и естественно переключается: подрос дракон → в бой/босс; яйцо
дозрело → в Гнездо; открылась кузница → улучшения; регион изучен → следующий. Гейтинг механик по уровню
разносит открытия во времени.

## 7. Рекомендации по дальнейшей интеграции

1. **Осколки → кузница**: 🔮 как ускоритель/скидка ковки — второй сток, связь с артефактами.
2. **Роли под активность**: Охотник → больше яиц в полёте; Поддержка → быстрее инкубация в логове;
   Контроллер → выше шанс редких событий. Драконы ↔ все активности.
3. **Задания-мосты**: ежедневки, требующие 2 системы («победи босса И согрей яйцо»).
4. **Скрытые области ↔ достижения**: открытие секретной локации — веха, дающая уникальное яйцо.
5. **Экономика валют**: ⭐звёзды владык — сейчас только Восхождение; добавить второй сток (обмен на редкий
   материал), чтобы валюта работала в 2+ системах.
6. **Карта мира как граф**: подписать на карте, какая победа/находка открывает следующий регион/механику.
