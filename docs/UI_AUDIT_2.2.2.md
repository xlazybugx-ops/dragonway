# Draconis 2.2.2-test - аудит интерфейса

Дата: 15 июля 2026. Объект проверки: мобильная веб-сборка для iPhone, экран хаба, исследование, пошаговый бой и глобальная навигация.

## Итог

Критические дефекты из контрольного снимка устранены. Причиной нечитаемой кириллицы был не только выбор гарнитуры: статический текст `index.html` был поврежден повторным преобразованием кодировки. Текст восстановлен в UTF-8, а единый стек интерфейса закреплен как Aptos с безопасными системными заменами. Пять разделов навигации теперь всегда делят доступную ширину поровну.

## Исправлено

- Восстановлены русские подписи верхней панели, навигации и стартового экрана; документ сохранен в UTF-8.
- Aptos назначен всем элементам интерфейса. Если Aptos отсутствует в iOS, используется Segoe UI или Arial без потери кириллицы.
- Источник спрайтов исследования и пошагового боя заменен: вместо полноэкранных `fly_*` используются прозрачные `arcade_*` для огня, льда, леса, шторма и тени.
- Нижняя навигация переведена на сетку из пяти равных колонок с ограничением ширины, многоточием и адаптивной подписью 8-10.5 px.
- Меню задач хаба превращено в жестовую шторку. Свайп вниз открывает квесты, активные бонусы и подсказку; свайп вверх закрывает.
- Новые задачи показываются при первом открытии хаба 4.2 секунды, затем доступны в шторке.
- Над навигацией добавлена компактная панель «Сокровища / Украшения / Рынок».
- Панель следующего шага поднята выше сервисной панели, чтобы элементы не перекрывались.

## Результаты аудита

### Критические - закрыты

1. Поврежденная кириллица делала основные разделы неразличимыми.
2. Непрозрачные изображения драконов закрывали карту и поле боя.
3. Пятый пункт навигации выходил за правый край на экране 945 px в логических координатах снимка.

### Высокий приоритет - закрыты

1. Задачи занимали постоянную вертикальную область и конкурировали с хабом.
2. Сервисы хаба были разнесены по верхней части сцены и хуже читались как единая группа.
3. Не было единого поведения открытия и закрытия панели задач жестом.

### Остаточные риски

- Aptos не входит в стандартный набор iOS. Для абсолютно одинаковой типографики на всех устройствах нужна лицензированная локальная поставка WOFF2; текущий системный fallback функционально безопасен.
- Подписи навигации на экранах уже 360 CSS px остаются очень компактными. После теста на реальных iPhone рекомендуется сократить «Странствие» до «Полет» или показывать подпись только активной вкладки.
- Часть вторичных меток имеет размер 9-10 px. Они не являются основными действиями, но должны проверяться на Dynamic Type и при пониженной остроте зрения.

## Матрица приемочного теста

- iPhone SE: 375 x 667 CSS px, портрет, увеличенный текст 100% и 115%.
- iPhone 13/14: 390 x 844 CSS px, safe area и жестовая навигация.
- iPhone 15 Pro Max: 430 x 932 CSS px.
- Проверки: все пять вкладок видны; кириллица читается; шторка не блокирует системный свайп; сервисная панель не перекрывает задания; прозрачность спрайтов сохраняется.

## Вердикт

Интерфейс готов к тестовому релизу при обязательном прогоне на трех размерах iPhone. Блокирующих интерфейсных дефектов по статическому и кодовому аудиту не осталось.
