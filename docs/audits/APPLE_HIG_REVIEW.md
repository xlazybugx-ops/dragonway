# APPLE_HIG_REVIEW.md — соответствие Apple HIG

Оценка против рекомендаций Apple для мобильных интерфейсов (по коду/CSS).

| Пункт HIG | Статус | Доказательство / замечание |
|---|---|---|
| Размер целей касания (≥44pt) | ✅ | таббар `min-height:56px`; медиазапрос `pointer:coarse` поднимает кнопки до 44px; кнопки навыков/плитки крупные |
| Отступы/поля | 🟡 | базовые паддинги есть; на iPhone SE (4.7") плотные экраны (Шпиль/Гнездилище) требуют проверки |
| Нижняя навигация (Tab Bar) | ✅ | `#tabbar` фиксирован снизу, 5 разделов, active-состояние, бейджи |
| Bottom Sheet | ✅ | бот-шит «Стая и Заповедник» (`#flockSheet`), анимация `popIn`, закрытие тапом по фону |
| Контекстные меню | 🟡 | долгое нажатие для «избранного» в Заповеднике есть; системного контекстного меню нет (не требуется) |
| Жесты | 🟡 | джойстик, тап-таргет, свайп-карусель; долгое нажатие. Свайпов «назад» между разделами нет |
| Безопасные зоны (safe-area) | ✅ | `env(safe-area-inset-bottom)` в таббаре и `#app`; НО топбар без `safe-area-inset-top` — проверить вырез/Dynamic Island |
| Динамический текст | 🟡 | есть режим «крупный текст» (a11y), но не завязан на системный Dynamic Type |
| Одноручное использование | ✅ | основные действия в нижней зоне (таббар, навыки, бот-шиты) |
| Уважение к reduced-motion | ✅ | `@media (prefers-reduced-motion:reduce)` отключает трансформы/вспышки |

## Замечания
1. **Топбар без `safe-area-inset-top`** — на устройствах с вырезом/Dynamic Island ресурсы вверху могут
   уходить под вырез. **P2.** Проверить и добавить верхний inset.
2. **Динамический текст** не привязан к системному Dynamic Type (только собственный тумблер). **P3.**
3. Нет системного жеста «назад» между разделами (только таббар). **P3.**
4. Плотность на 4.7" — проверить отступы. **P2.**

## Вывод
Базовые принципы HIG соблюдены хорошо (таббар, bottom sheet, 44px, safe-area снизу, reduced-motion).
Открытые вопросы — верхняя безопасная зона и плотность на малых экранах; проверяются на устройстве.
