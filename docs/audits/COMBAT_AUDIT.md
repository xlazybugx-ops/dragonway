# COMBAT_AUDIT.md — аудит боевой системы (независимый)

Две системы: пошаговая (`08-battle.js`, 1×1) и аркадная реального времени (`08b-arcade.js`, 1×стая).

| Критерий | Пошаговый | Аркада |
|---|---|---|
| Читаемость | ✅ HP/мана барами+числами, оценка урона lo–hi, стихия ▲/▽ | ✅ минимальный HUD, счётчик врагов |
| Зоны поражения | н/д (1×1) | ✅ предпросмотр линия/круг/радиус/цепочка + 🎯N до подтверждения |
| Очередь ходов | ✅ полоса «Ты → Соперник» | н/д (реальное время, телеграфы врагов) |
| Массовые способности | н/д | ✅ AoE подсвечивает всех в радиусе |
| Одиночные способности | ✅ единственная цель + маркер ⌖ | ✅ цель + линия атаки |
| Визуальные эффекты | ✅ shake/lunge, всплывающие числа | ✅ частицы, вспышки, тряска |
| Обратная связь | ✅ лог + чипы эффектов + свечение готовой ульты | ✅ fx-зоны появляются/гаснут |
| Ощущение попаданий | ✅ звук hit/crit + вибро + анимация | ✅ вспышка цели + частицы |
| ИИ противников | ✅ FSM боссов (фазы/щит/ярость/застыл), роли | 🟡 базовый: сближение + телеграф-удар |
| Скорость боя | ✅ мини-игра тайминга, регулируемая | ✅ динамичная |

## Баланс (из GB, не менялся)
Нелинейная формула `0.58·atk²/(atk+def)·pow`; стихии ×1.28/×0.85; крит ×1.5; тайминг 0.85–1.25; комбо
до ×1.15; босс-слабость ×1.28. Значения выглядят согласованно; калибровка ранее проверялась
симуляцией (30–60с нормальный бой). **Живой баланс на устройстве не проверялся.**

## Слабые места
1. **ИИ аркадных рядовых примитивен** (сближение+удар) — на длинной дистанции однообразен. **P2.**
2. Пошаговый бой 1×1 — нет массовых способностей как класса (по дизайну). Не проблема, но ограничивает
   тактическую глубину.
3. Баланс подтверждён только симуляцией/статикой — нужна живая проверка. **P1 (QA).**

## Вывод
Боевая читаемость — сильная сторона проекта (очередь/эффекты/цель/готовность ульты/предпросмотр зон).
Главный пробел — примитивный ИИ аркадных врагов и отсутствие живой балансной проверки.
