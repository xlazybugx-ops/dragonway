# PERFORMANCE_AUDIT.md — аудит производительности (независимый)

Оценка по коду. Живых замеров FPS/памяти на устройстве НЕТ — это обязательный пункт QA беты.

## Изображения — главный риск
- `images/` = **~25 МБ**, из них **фоны полёта ~20 МБ** (15 файлов `fly_*.webp` по 1.3–2.1 МБ).
- Загрузка **on-demand**: `loadFlyMap()` тянет фон только при входе в ярус биома, есть fallback на
  процедурный фон при `onerror`. Это снижает первичный вес, НО каждый ярус = 1.3–2.1 МБ на сотовой
  сети → заметная задержка/трафик. **P1.**
- `den_*.png` — 5 файлов 0.55–0.81 МБ. Спрайты видов `*_{1,25,60,100}.webp` — компактны (12–35 КБ).
- Нет `loading="lazy"` на большинстве `<img>` (они и так рендерятся по требованию через JS `Image()`).

## rAF-циклы и таймеры
- Три rAF-сцены: живое логово (05), полёт (07), аркада (08b). Каждая:
  - самоостанавливается, когда её вид не `.on` (проверка в кадре) — ✅ хорошо;
  - dt-clamp (`Math.min(0.05,…)`) и try/catch в кадре — ✅ устойчиво к фризам/сбоям.
- Слушатель resize снимается при остановке сцены (`_lairFit` cleanup) — ✅ без накопления.

## DOM и утечки
- HUD-элементы кэшируются (flight `f._score/_stam`, arcade `arc._hud/_abBtns`) — ✅ без `$()` в кадре.
- Обработчики боя/полёта вешаются на `AbortController`-signal и window-события с явным `removeEventListener`
  в `arcFinish` — ✅ снимаются.
- Переиспользуемые узлы (`#lvlFlash`, `#eggCeremony`, `#toast`, `#float`) — не плодятся. ✅
- Частицы/зоны — массивы с фильтрацией по времени жизни (пулов нет, но объёмы малы). 🟡 приемлемо.

## Анимации
- Микроанимации на CSS-переходах (GPU), 150–200мс, с `prefers-reduced-motion`. ✅

## Safari iPhone
- Кэш-бастинг `?v=33`; meta no-cache. WebAudio с `resume()` по политике браузера. safe-area в таббаре.
- **Не проверено на реальном устройстве** — обязательный пункт беты (FPS в полёте/аркаде/логове,
  память при длительной сессии, поведение на слабой сети).

## Риски
1. **~20 МБ фонов полёта** — трафик/задержки на сотовой. **P1.** Исправление: пережать webp
   (target ≤400–600 КБ/файл), адаптивные размеры, прелоад следующего яруса. Сложность: низкая-средняя.
2. Отсутствие живых замеров — **P1 (QA).**
3. Нет пула частиц — **P3** (при экстремальной плотности).

## Вывод
Архитектура рантайма аккуратная (самоостановка циклов, кэш DOM, снятие слушателей, try/catch). Главный
измеримый риск — вес графики полёта и отсутствие проверки на устройстве.
