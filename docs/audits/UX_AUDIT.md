# UX_AUDIT.md — аудит экранов (независимый)

Оценка каждого экрана по 11 критериям, итог 1–10 (строго, по планке AAA-mobile). Основано на коде
(разметка `index.html`, рендеры в `05/06/08/10`, стили `style.css`).

Критерии: понятность · простота · визуальная нагрузка · скорость частых действий · одна рука ·
навигация · инфоперегруз · объём текста · читаемость · использование пространства · соответствие
мобильным играм.

## Системные наблюдения

- **Навигация:** единая нижняя панель (5 табов, ≥44px, active-состояние) — сильное решение. Но
  первый экран (Логово=hub) использует «кликабельные здания» вместо явных кнопок — двойная модель
  навигации (таббар + сцена) может путать.
- **Инфоперегруз:** после реворков большинство второстепенных данных ушло под спойлеры/бот-шиты.
  Остаточная плотность — карта странствий и деревья (Шпиль/Гнездилище).
- **Текст:** вводные `lede` приглушены, но всё ещё присутствуют почти на каждом экране (в
  `index.html` захардкожены длинные абзацы-описания разделов).
- **Одна рука:** таббар, бот-шиты, кнопки навыков — в нижней зоне; цели касания ≥44px (медиазапрос
  `pointer:coarse`). В целом — хорошо.
- **Читаемость:** есть режимы доступности (контраст/крупный текст/левши/без цвета).

## Проблемы UX (сводно)

1. Двойная модель навигации на хабе (таббар + клик-зоны) — **P2**.
2. Плотные экраны Шпиль/Гнездилище/Кузница на малых экранах — **P2**.
3. Длинные захардкоженные `lede`-описания в `index.html` — **P3**.
4. Карта странствий текстово плотная — **P3**.

## Рекомендации

- Свести хаб к явным крупным кнопкам-разделам или убрать дублирование с таббаром.
- Свернуть длинные описания в раскрываемые панели; в кодексе — «главное → детали по тапу».
- Ревизия плотности деревьев на 4.7" (iPhone SE).
