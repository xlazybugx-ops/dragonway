# UX_MINIMAL.md — минималистичный интерфейс

Главное правило: **сначала графика → потом цифры → потом текст.**
Иконка, цвет, бар и анимация несут смысл; текст появляется только когда действительно нужен.

---

## 1. Иерархия подачи информации

| Приоритет | Средство | Примеры в игре |
|---|---|---|
| 1. Графика | силуэт дракона, эмодзи-сигил, окрас морфа, живое логово | карточка дракона, аркада |
| 2. Цвет | стихия/редкость/состояние (жизнь, мана, урон, крит, легенда) | бары, звёзды редкости, тосты |
| 3. Индикаторы | прогресс-бары, счётчики, кулдаун-заливка, значки эффектов | HP/XP/инкубация/навыки |
| 4. Текст | только смысловая нагрузка: имя, число, короткая цель | под спойлером/по тапу |

---

## 2. Что сделано

**Приглушение вводного текста.** `.lede` (пояснительные абзацы панелей) переведён в тихий стиль
(меньше, тусклее) — он больше не конкурирует с игровыми элементами за внимание.

**Панель уровня логова = графика+цифры, без предложений.** Вместо описания: `🏰 ур.3 · 8/8🐉 · ⬆️`.
Всё считывается за долю секунды: иконка замка, число-вместимость (краснеет при заполнении),
кнопка улучшения с целевой цифрой.

**Резерв под спойлером.** Драконы вне логова спрятаны в `<details> 💤 Резерв · N`. По умолчанию
экран чистый; список открывается по тапу. Каждый резервный дракон — компактная карточка
(эмодзи + имя + уровень + «↩ логово»), а не строка текста.

**Компактные карточки вместо блоков.** Добавлены утилити-классы `.ux-card` (иконка + заголовок +
подпись) и `details.ux-more` (скрываемая панель) для замены длинных текстовых блоков.

---

## 3. Скрываемые панели (press / tap-to-reveal)

Паттерн `details.ux-more`:
```html
<details class="ux-more"><summary>Подробнее</summary> …длинный текст… </details>
```
По умолчанию свёрнуто → интерфейс чистый. Стрелка ▾/▴ подсказывает раскрытие. Кодекс следует
тому же принципу: в списке — только главное (сигил, имя, редкость), полное описание открывается
на карточке отдельно.

---

## 4. Бой: только необходимое

Во время аркадного боя на экране остаётся: **HP** (шкала), **щит** (наложение на HP),
**энергия/мана** (шкала), **значки эффектов** над врагами (🔥/🟢/❄️), **готовность навыков**
(заливка кулдауна на кнопках), **счётчик врагов** (в логове), **текущая цель** (подсветка + линия
атаки). Второстепенный текст убран — числа урона всплывают кратко и гаснут.

Полная модель читаемости боя — в `COMBAT_READABILITY.md`.

---

## 5. Эргономика (сохранена и учтена)

Цели касания ≥ 44px (`@media pointer:coarse`), кнопки навыков и резерва в зоне большого пальца
(низ/право), тосты — очередь по приоритету. Узкие экраны (≤380px) прячут дублирующие подписи
`.ux-hide-narrow` — иконка и цвет сохраняют смысл.

## 7. Рекомендации

1. Постепенно обернуть длинные `.lede` панелей в `details.ux-more` (лор — по желанию, не на старте).
2. Свести кодекс к сетке иконок с модалкой-деталями (карточка по тапу).
3. Ввести единый цветовой словарь состояний (жизнь/мана/урон/крит/легенда) и применять его везде.
4. Заменить оставшиеся текстовые статус-строки на бар+иконку, где возможно.
5. Добавить настройку «плотный/просторный» режим интерфейса для разных размеров экрана.
